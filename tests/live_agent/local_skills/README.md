# Local Filesystem Skill Test

This suite verifies the normal local Skill delivery path used by `codex exec`.
Codex discovers metadata from a filesystem-backed `.agents/skills/**/SKILL.md`,
renders the enabled Skill catalog into the model context, and injects the full
Skill body when the prompt explicitly names `$local-skill-fixture`.

This is not a `skills` Namespace test. Local Skill discovery and explicit
injection do not require, call, or emulate `skills.list` or `skills.read`.

## Scenario

- `01`: explicitly select the copied `local-skill-fixture`; the exact success
  marker exists only in its `SKILL.md`, not in `TASK.md` or its catalog
  description.

## Required runner and configuration

Use the normal isolated local-mode `codex exec` runner with its local execution
environment attached. Copying the task must preserve the hidden
`.agents/skills/local-skill-fixture/SKILL.md` path under the worktree. Use
Provider ID `codex_rosetta`, display name exactly `OpenAI`, and the catalog
generated by local mode.

The run must prove all of the following separately:

1. the developer Skill catalog includes the fixture name, description, and
   filesystem source locator;
2. the explicit `$local-skill-fixture` selection causes Codex to read and
   inject the complete body as a Skill input fragment;
3. the model returns the body-only marker without calling `skills.list`,
   `skills.read`, shell, or file tools.

An absent catalog entry or absent explicit injection is an exposure failure.
Do not replace either one with an orchestrator provider or app-server runner.
