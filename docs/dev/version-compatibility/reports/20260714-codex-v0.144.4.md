## Reviewed source changes

The ordinary built-in tool registry and the bundled eight-model JSON are
unchanged between 0.144.1 and 0.144.4. Rosetta therefore advances the static
tool-catalog source binding without changing its tool entries, and renames the
byte-identical packaged model resource to bind it explicitly to 0.144.4.

The extracted source contract has two semantic changes:

- `/realtime/calls` is no longer present in the reviewed endpoint constants, so
  `REALTIME_CALLS_ENDPOINT` was removed from the baseline;
- `ModelMessages` now accepts optional `auto_review`, whose
  `AutoReviewMessages.policy` supplies auto-review policy guidance. This nested
  field was added to the baseline and model-catalog reference.

Guardian/automatic-review assembly changed outside the ordinary static tool
inventory. Codex no longer unconditionally forces the older direct-only review
tool plan; the selected review model's catalog `tool_mode` can now choose Code
Mode. This is a behavior change even though the official `codex-auto-review`
JSON entry itself is unchanged and keeps `tool_mode: null`.

## Rosetta adaptation

Local mode preserves the official `codex-auto-review` entry when its upstream
model is omitted, empty, or also named `codex-auto-review`. This keeps the more
native Responses behavior available when the upstream is OpenAI or a GPT relay
that accepts the original request shape.

When the exposed `codex-auto-review` alias explicitly maps to another upstream
model name, Rosetta changes only that generated entry's `tool_mode` to
`code_mode_only`. The mapping is the user-provided signal for a non-OpenAI
review service; narrowing the client to the newer Code Mode avoids requiring
Rosetta to maintain every legacy and mixed Guardian tool surface. Rosetta does
not infer this decision from provider names, endpoints, or URLs.

Automated coverage includes object and shorthand-string model configuration,
same-name/no-name preservation, different-upstream forcing, bundled-resource
identity, and static catalog metadata. Real review-model passthrough and mapped
third-party review execution remain triggered live gates.
